# Music Recommendation MLOps 🎵

End-to-end content-based music recommendation system with reproducible data/model pipelines, MLflow tracking and registry, FastAPI serving, Dockerized deployment, and GitHub Actions CI/CD automation.

## 1. Problem Statement 🎯

Goal: recommend songs similar to a user's current taste based on audio features.

Design priorities:
- Reproducibility
- Explainability
- Production-readiness
- Modular MLOps pipeline

This project intentionally uses content-based filtering instead of collaborative filtering.

## 2. Theoretical Foundation 📚

### 2.1 Content-Based Filtering 🧠

Each song is represented as a numeric feature vector in an audio-feature space. Recommendations are generated by finding nearest songs to a query vector.

Why content-based here:
- No user-user interaction matrix required.
- Works even for new users with short session history.
- Feature-level explainability is straightforward.

### 2.2 Similarity with Cosine Distance 📐

Similarity is computed with cosine geometry. For vectors $a$ and $b$:

$$
	ext{cosine\_similarity}(a,b)=\frac{a\cdot b}{\|a\|\|b\|}
$$

Cosine distance used by KNN is:

$$
	ext{cosine\_distance}(a,b)=1-\text{cosine\_similarity}(a,b)
$$

Why cosine:
- Focuses on direction, not absolute magnitude.
- Effective for standardized feature vectors.

### 2.3 Feature Standardization ⚖️

Features are scaled with StandardScaler:

$$
z = \frac{x-\mu}{\sigma}
$$

This prevents large-range features from dominating distance calculations.

### 2.4 Session Profile with Exponential Decay 🔁

User profile vector is a weighted average of session history vectors.

For history size $n$, oldest item gets weight $\text{decay}^{n-1}$ and newest gets weight $1$. With decay $0.85$, recent songs influence recommendations more.

## 6. DVC Pipeline 🔄

Defined in dvc.yaml:

1. download
	- cmd: python src/data/download.py
	- out: data/raw/dataset.csv

2. preprocess
	- cmd: python src/data/preprocess.py
	- deps: raw dataset + preprocess script
	- params: feature_cols, drop_cols, n_features
	- outs: features.csv, scaler.pkl

3. train
	- cmd: python src/model/train.py
	- deps: features.csv + train script
	- params: n_neighbors, metric, algorithm, decay, n_features, feature_cols, dataset_version, eval_sample_size
	- out: knn_model.pkl

Current params in params.yaml:
- n_neighbors: 10
- metric: cosine
- algorithm: brute
- decay: 0.85
- n_features: 8
- dataset_version: v1
- eval_sample_size: 100

## 9. FastAPI Service 🚀

Main app: api/main.py

Startup behavior:
- Load recommender model.
- Load processed features and metadata.
- Load scaler.
- Initialize session store with 30-minute TTL.

### 9.1 Endpoints

1. GET /health
	- Returns API status, model loaded state, source/version, song count.

2. GET /search
	- Query param: q
	- Case-insensitive partial search on track_name.
	- Returns up to 25 matches.

3. POST /session/start
	- Input: song_index
	- Starts session and returns first recommendations.

4. POST /session/{session_id}/next
	- Input: song_index and action in {play, skip, replay}
	- Updates profile and returns refreshed recommendations.

5. GET /session/{session_id}/history
	- Returns played songs, action timeline, current profile vector.

6. DELETE /session/{session_id}
	- Deletes session from memory.

### 9.2 Session Behavior

- play: adds song vector
- replay: adds weighted vector (1.5x)
- skip: no vector update
- profile_shift: cosine distance between old and new profile vectors

## 14. Troubleshooting 🩺

### 14.1 DVC error: output tracked by SCM ⚠️
Symptom:
- output data/raw/dataset.csv is already tracked by SCM

Cause:
- DVC outputs still tracked in Git index.

Fix:

```bash
git rm -r --cached -- data/raw/dataset.csv data/processed/features.csv data/processed/scaler.pkl data/processed/knn_model.pkl
git commit -m "stop tracking DVC outputs"
dvc repro
```

### 14.2 Docker workflow fails with login error 🔐
Symptom:
- Username and password required

Fix:
- Ensure GitHub repository secrets are named exactly:
  - DOCKERHUB_USERNAME
  - DOCKERHUB_TOKEN

### 14.3 Retrain shows zero runs
This is expected until manually triggered because it uses workflow_dispatch only.
